# User Stories — methodology

## Purpose

Generate user stories from requirements using "As a [user], I want [goal], So that [benefit]" format.

## Stance

- **User perspective, not system perspective.**
- **One story = one user benefit.**
- **Acceptance criteria are testable.** Given/When/Then format.

## User Story Format

```
As a [user type],
I want to [goal],
So that [benefit].
```

## Acceptance Criteria Format

```
Given [precondition],
When [action],
Then [expected outcome].
```

## Required Sections

### User Stories (with priority)
- P0: Must have
- P1: Should have
- P2: Nice to have

### Acceptance Criteria per story
- 2-5 criteria per story
- Each criterion testable

### Out of Scope
- Things explicitly not in this change
