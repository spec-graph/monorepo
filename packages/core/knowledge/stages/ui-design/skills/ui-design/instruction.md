# UI Design — methodology

## Purpose

Design user interfaces (wireframes, component tree, design system) before implementation.

## Stance

- **Design from user perspective.** What does the user see and do?
- **Reuse design system components.**
- **Accessibility is not optional.** WCAG compliance from day one.

## Required Sections

### User flows
- Step-by-step interactions
- Decision points
- Error / Empty / Loading states

### Component tree
- Atomic → Molecular → Organism → Page

### Design tokens
- Color, Typography, Spacing, Border, Shadow

### Accessibility
- WCAG AA contrast (≥ 4.5:1)
- Keyboard navigation
- Screen reader (ARIA)
- Focus management

### Responsive design
- Mobile (320-768px)
- Tablet (768-1024px)
- Desktop (1024+)
