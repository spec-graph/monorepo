# Requirement Analysis Pack Context

## When This Pack Activates
- Always (every project needs requirements analysis)
- Profile dimension `field` = `greenfield` or `brownfield`

## What This Pack Provides
- Product Brief template
- Impact Map template
- PRD (Product Requirements Document) template
- Story Map template

## Key Artifacts
- `requirement/product-brief` — One-page product summary
- `requirement/impact-map` — Impact mapping (Why → Who → What → How)
- `requirement/prd` — Full PRD with user stories and AC
- `requirement/story-map` — Story map for release planning

## Agent Guidance
- Requirements must be testable (Given/When/Then format)
- Impact mapping ensures every requirement traces to a business goal
- Story map organizes stories by user activity (horizontal) and priority (vertical)
- Brownfield projects: start with impact-map to understand what exists
