## 7. 优先级矩阵 (Priority Matrix)

```
                    影响大
                      ▲
                      │
  高价值低风险    │  高价值高风险
                │
(P2) ───────────┼────────── (P0)
                │
  低价值低风险    │  低价值高风险
                      │
                      ▼
                    影响小

          ← 风险低        风险高 →
```

| 优先级 | 投入策略        | 典型特征                          |
| ------ | --------------- | --------------------------------- |
| **P0** | 必须做,先做     | 高影响低风险 → 快速交付赢取时间   |
| **P1** | 必须做,安排资源 | 高影响高风险 → 先做 Spike 验证    |
| **P2** | 可选做,看资源   | 低影响低风险 → 有空再做           |
| **P3** | 不做或延后      | 低影响高风险 → 放弃或寻找替代方案 |

## 模板使用说明

> **影响地图四问法**:
>
> 1. **Why** → 为什么要做？(目标)
> 2. **Who** → 谁会产生影响？(角色)
> 3. **How** → 他们的行为如何改变？(影响)
> 4. **What** → 我们要交付什么来产生这个改变？(交付物)
>
> **适用场景**:
>
> - 项目启动初期,对齐目标与范围
> - 需求优先级争论时,回到"为什么"
> - Sprint 规划时,确保 Story 都关联到业务目标
> - 向管理层汇报时,展示"交付物 → 影响 → 目标"的链路
>
> **核心价值**: 防止"为做功能而做功能",确保每一行代码都服务于可测量的业务目标。
