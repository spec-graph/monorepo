## 3. 组件级设计

### 3.1 导航栏 (Header)

```
┌─────────────────────────────────────────────────────────────┐
│ 📦 Logo          首页 / Dashboard       [🔍 搜索]  [👤 用户] │
│                                                                │
│  [项目 ▾] [功能1]  [功能2]  [功能3]  [⚙️ 设置]               │
└─────────────────────────────────────────────────────────────┘
```

| 元素     | 交互             | 状态                         |
| -------- | ---------------- | ---------------------------- |
| Logo     | 点击跳转到首页   | Default / Hover              |
| 导航项   | 点击切换功能模块 | Active / Inactive / Disabled |
| 搜索     | 点击展开搜索框   | Collapsed / Expanded         |
| 用户菜单 | 点击展开下拉菜单 | Closed / Open                |

### 3.2 内容卡片 (Card)

```
┌─────────────────────────────┐
│ 🖼️ 封面图 (可选)            │
│                             │
│ 卡片标题                      │
│ 卡片描述文本 (最多 2 行)     │
│                             │
│ [主要操作]  [次要操作]       │
└─────────────────────────────┘
```

**变体**:

- **基础卡片**: 标题 + 描述 + 操作按钮
- **媒体卡片**: 带封面图 + 标题 + 描述
- **数据卡片**: 统计数值 + 趋势图 + 对比数据
- **空状态卡片**: 空状态插图 + 引导文案

### 3.3 列表项 (List Item)

```
┌─────────────────────────────────────────────────────────────┐
│ [ ]  列表项标题                          标签  [操作按钮]  │
│      辅助说明文本 (灰色小字)                                 │
└─────────────────────────────────────────────────────────────┘
```

**状态**:

- Default: 正常显示
- Hover: 背景高亮,显示更多操作
- Selected: 选中状态 (左侧蓝色竖条)
- Disabled: 灰化,不可点击
- Loading: Skeleton 骨架屏

### 3.4 表单 (Form)

```
┌─────────────────────────────────────────────────────────────┐
│ 表单标题                                                      │
│ 表单说明副标题                                                │
│                                                             │
│  字段 1 (必填) *                                             │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ 占位符提示文本                                         │  │
│  └─────────────────────────────────────────────────────┘  │
│  字段 1 帮助说明文本 (灰色小字)                              │
│                                                             │
│  字段 2 (可选)                                              │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ 占位符提示文本                                         │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                             │
│  [取消]                           [重置]  [提交]            │
└─────────────────────────────────────────────────────────────┘
```

**表单验证状态**:

- ✅ 验证通过: 绿色边框 + 成功图标
- ❌ 验证失败: 红色边框 + 错误提示文本
- ⚠️ 警告: 黄色边框 + 警告提示
- ⏳ 验证中: 加载状态

## 4. 响应式适配

### 4.1 断点定义

| 断点    | 设备类型 | 宽度范围       | 布局变化             |
| ------- | -------- | -------------- | -------------------- |
| Mobile  | 手机     | < 768px        | 单列布局,汉堡菜单    |
| Tablet  | 平板     | 768px - 1024px | 双列布局,简化导航    |
| Desktop | 桌面     | ≥ 1024px       | 三/四列布局,完整导航 |

### 4.2 移动端适配

```
手机竖屏 (375px):

┌──────────────────┐
│ 📦    [🔍] [👤]  │
├──────────────────┤
│  页面标题        │
│  [主按钮]        │
│                  │
│ ┌─────────────┐ │
│ │ 卡片 1      │ │
│ └─────────────┘ │
│ ┌─────────────┐ │
│ │ 卡片 2      │ │
│ └─────────────┘ │
│                  │
│  [+] 悬浮按钮    │
└──────────────────┘
```

**移动端特殊适配**:

- 点击目标 ≥ 44px × 44px (防止误触)
- 表单标签在输入框上方,不做左右布局
- 按钮全宽,不左右排列
- 复杂表格改为"卡片式列表"展示

## 5. 交互与动效

### 5.1 微交互

| 交互元素 | 触发方式 | 动效描述                      | 时长  |
| -------- | -------- | ----------------------------- | ----- |
| 按钮     | Hover    | 背景色渐变 + 轻微放大         | 150ms |
| 按钮     | Click    | 按压效果 (缩小 2%) + 波纹扩散 | 300ms |
| 卡片     | Hover    | 上移 4px + 阴影加深           | 200ms |
| 模态框   | 打开     | 从中心缩放 + 背景模糊         | 300ms |
| 列表项   | 删除     | 向左滑出 + 下方项上移填充     | 250ms |

### 5.2 加载状态

| 场景         | 加载方式                         | 超时处理                 |
| ------------ | -------------------------------- | ------------------------ |
| 页面初始加载 | 顶部 Progress Bar                | 10s 超时 → 错误页 + 重试 |
| 按钮点击     | 按钮内 Loading Spinner           | 禁用按钮,防止重复提交    |
| 列表加载     | Skeleton 骨架屏 (与实际布局一致) | 5s 后显示"加载缓慢"提示  |
| 图片加载     | 模糊占位图 → 逐渐清晰            | 加载失败 → 默认占位图    |

### 5.3 空状态

```
┌─────────────────────────────────────────┐
│                                         │
│          📭  还没有任何数据              │
│                                         │
│   创建第一个项目,开始你的旅程吧！          │
│                                         │
│           [ 创建项目 ]                   │
│                                         │
└─────────────────────────────────────────┘
```

**空状态原则**:

1. 明确告知"这里是什么",不要让用户猜测
2. 说明"为什么是空的"
3. 给出"下一步可以做什么"的引导
4. 友好的插图,减轻挫败感

## 6. 可访问性规范

### 6.1 键盘操作

| 按键        | 行为                             |
| ----------- | -------------------------------- |
| Tab         | 按逻辑顺序在可交互元素间移动焦点 |
| Shift + Tab | 反向移动焦点                     |
| Enter       | 激活当前聚焦元素 (点击/提交)     |
| Space       | 触发按钮/复选框/单选框           |
| Esc         | 关闭模态框/下拉菜单/返回上一级   |
| ↑ ↓ ← →     | 在列表/表格/分页中导航           |

### 6.2 焦点样式

- 焦点环: 2px solid #2563eb + 2px 偏移
- 对比度 ≥ 3:1
- 不会被其他元素遮挡

### 6.3 颜色使用

- 不只用颜色传达状态,必须配合图标/文字
- 例如: 错误 = ❌ 红色图标 + "错误"文字
- 例如: 成功 = ✅ 绿色图标 + "成功"文字

## 7. 验收标准

- [ ] **信息架构**: 层级清晰,用户 3 秒内知道"我在哪、能做什么"
- [ ] **响应式**: 所有断点下布局不破坏,内容不溢出
- [ ] **可访问性**: 键盘可操作,颜色有足够对比度,屏幕阅读器友好
- [ ] **交互反馈**: 所有操作 ≤ 100ms 给出反馈,加载状态不困惑
- [ ] **空状态**: 所有列表/内容区都有合理的空状态引导
- [ ] **错误处理**: 所有可能出错的场景都有友好提示 + 恢复路径
- [ ] **一致性**: 与设计系统组件样式/交互一致,不出现特例

## 模板使用说明

> **什么时候画线框图?**
>
> - 需求确定后,高保真设计之前
> - 快速验证信息架构与布局合理性
> - 跨团队讨论交互方案,低成本试错
>
> **原则**:
>
> - **低保真,高思考**: 不纠结视觉细节,重点在布局层级与用户路径
> - **灰阶+矩形**: 用线条/方框/占位符表达,不用真实图片/颜色
> - **标注优先**: 关键交互/状态/异常场景用文字标注清楚,不要让看的人猜
> - **移动端优先**: 先想手机怎么摆,再扩展到大屏
