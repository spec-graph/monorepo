# Product Brief: spec-graph

## 产品概述

**spec-graph** 是一个领域中立的规格驱动图编排内核，为 AI 辅助开发提供结构化的工作流管理能力。它将 AI 生成代码的过程从"凭记忆做"转变为"声明式 + 状态机驱动"，确保开发流程的可追踪性、可审计性和可复现性。

## 核心价值

### 1. 结构化工作流
- 将模糊的 AI 开发流程转化为明确的阶段和动作
- 通过状态机确保流程的正确性和一致性
- 支持复杂的多阶段工作流编排

### 2. 完整追溯性
- 建立从需求到代码的完整追溯链
- 支持影响分析和变更管理
- 满足合规和审计要求

### 3. 质量内建
- 通过 gate 机制在关键节点强制质量检查
- 通过 checklist 验证文档质量
- 防止不合格的产出进入下一阶段

### 4. 领域中立
- 内核不包含任何领域特定概念
- 通过 pack 系统支持任意领域
- 一套引擎，多种应用

## 目标用户

### 1. AI Agent (Claude/Codex)
- 需要清晰的工作流指导
- 需要自动化的状态追踪
- 需要完整的文档生成支持

### 2. 开发者
- 需要可视化的工作流状态
- 需要快速的问题定位能力
- 需要便捷的文档管理工具

### 3. 项目经理
- 需要项目进度概览
- 需要质量指标监控
- 需要风险预警机制

## 核心功能

### 1. 工作流编排
- **初始化**: `spec-graph init` 分析项目并生成工作流图
- **调度**: `spec-graph dispatch` 获取下一步动作
- **执行**: `spec-graph run` 自动执行确定性动作
- **状态**: `spec-graph status` 查看当前状态

### 2. 文档管理
- **创建**: 通过模板生成 PRD、架构文档、Stories、Tasks
- **追踪**: 自动注册 artifacts 并追踪状态
- **验证**: checklist 验证文档质量

### 3. 追溯系统
- **建立**: 建立 artifact 间的追溯关系
- **查询**: 查询完整的追溯链
- **分析**: 分析变更的影响范围

### 4. 质量门控
- **定义**: 在 graph.yaml 中定义 gate 条件
- **评估**: 自动评估 gate 是否通过
- **阻断**: gate 失败时阻止状态转移

## 技术架构

### 三层架构
1. **声明式 Pack 层**: 领域知识以 pack 形式提供
2. **确定性 Compose 层**: 纯规则引擎合成工作流图
3. **强制 Enforce 层**: 状态机和 gate 强制质量

### 6 个核心原语
1. **Work-unit**: 有状态的工作节点
2. **Artifact**: 带类型的产物
3. **Contract**: 有 producer/consumer 双边的 typed Artifact
4. **Check**: 声明式校验
5. **Gate**: 状态转移上的不变量条件
6. **Trace-edge**: 任意节点间的 typed 边

### 技术栈
- **语言**: TypeScript 5.x
- **运行时**: Node.js 20+
- **测试**: Vitest (484 tests, 100% pass)
- **CLI**: Commander.js
