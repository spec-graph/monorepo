# Story: 调度下一步动作

## Story 概述

**作为** AI Agent(Claude/Codex),
**我希望** 运行 `spec-graph dispatch` 获取下一步动作,
**以便** 知道当前应该生成什么文档、运行什么检查、转移到哪个阶段。

## 关联 Epic

- EPIC-001: spec-graph 核心工作流引擎

## 验收标准

### AC-1: 获取下一步动作
**Given** 工作流未完成
**When** 运行 `dispatch --json`
**Then** 返回 actions[0] 包含 type/id/agent_id

### AC-2: 文档指导字段
**Given** 动作是 produce_artifact
**When** 返回 manifest
**Then** 包含 template_ref / suggested_doc_path / document_guidance

### AC-3: 可执行命令
**Given** 动作是 run_check
**When** 返回 manifest
**Then** 包含 check_command 可直接执行

### AC-4: Sub-agent 指示
**Given** 动作需要 sub-agent
**When** 返回 manifest
**Then** requires_sub_agent=true 并包含 agent_prompt_ref

### AC-5: 确定性动作
**Given** 动作是 transition
**When** 返回 manifest
**Then** requires_sub_agent=false, next_step 可执行

### AC-6: 失败详情
**Given** gate 未通过
**When** 返回 manifest
**Then** 包含 missing_artifacts/failed_checks/missing_traces/missing_contracts
