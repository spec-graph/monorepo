# Story: 初始化项目工作流

## Story 概述

**作为** 开发者,
**我希望** 能够用一行命令分析项目并生成工作流图,
**以便** 快速获得适配当前项目类型的流程定义。

## 关联 Epic

- EPIC-001: spec-graph 核心工作流引擎

## 需求引用

- PRD-001: spec-graph 工作流编排引擎 (US-1: 初始化项目工作流)

## 验收标准

### AC-1: 初始化命令
**Given** 一个新项目
**When** 运行 `spec-graph init`
**Then** 生成 `.spec-graph/` 目录和 profile.yaml

### AC-2: Sense 分析
**Given** 项目有 package.json
**When** sense 分析
**Then** 正确识别为 Node.js 项目,设置 facts.deployment=process

### AC-3: Compose 生成
**Given** profile.yaml 已生成
**When** 运行 `spec-graph compose`
**Then** 生成 graph.yaml 包含 actions/artifacts/checks/gates

### AC-4: Profile Override
**Given** 用户指定 `--build=api`
**When** 初始化
**Then** profile 中 has_ui=none, boundary=api

### AC-5: 幂等性
**Given** `.spec-graph/` 已存在
**When** 再次运行 `spec-graph init`
**Then** 提示已存在,除非使用 `--force`

## 技术实现要点

### 文件结构
```
.spec-graph/
├── profile.yaml          ← Sense 产物(9 维事实)
├── graph.yaml            ← Compose 产物(工作流图)
├── machine-state.yaml    ← 状态机(当前阶段/已完成 artifacts)
├── artifacts/<type>/     ← AI Agent 生成的文档
├── analysis/             ← 阶段分析元数据
└── changes/              ← 变更描述符
```

### 关键模块
- `src/commands/init.ts` - 初始化命令
- `src/commands/sense.ts` - Sense 命令
- `src/commands/compose.ts` - Compose 命令
- `src/engine/sense/index.ts` - Sense 引擎
- `src/engine/compose/index.ts` - Compose 引擎
