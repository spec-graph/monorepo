# Story: 追踪文档与状态

## Story 概述

**作为** 开发者,
**我希望** spec-graph 追踪每个阶段生成的文档和决策,
**以便** 能够回溯任何代码变更的来源。

## 关联 Epic

- EPIC-001: spec-graph 核心工作流引擎

## 验收标准

### AC-1: 记录文档路径
**Given** AI Agent 生成了 PRD 文档
**When** 运行 `spec-graph analysis --docs <path>`
**Then** 记录文档路径到 `.spec-graph/analysis/<phase>.yaml`

### AC-2: 查看阶段分析
**Given** 想查看 propose 阶段的分析
**When** 运行 `spec-graph analysis --phase propose`
**Then** 显示该阶段的关键决策、链接任务和文档

### AC-3: 文档质量检查
**Given** 想验证文档质量
**When** 运行 `spec-graph checklist <artifact-id>`
**Then** 返回机械检查和软检查结果

### AC-4: 链接追踪
**Given** 使用 `spec-graph analysis` 命令
**When** 指定 `--tasks` 和 `--artifacts`
**Then** 建立任务→文档→模板的追踪链

### AC-5: 状态更新
**Given** 文档生成完成
**When** 运行 `spec-graph artifact complete <artifact-id>`
**Then** 更新 machine-state.yaml 标记 artifact 为 completed
