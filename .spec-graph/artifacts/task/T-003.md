## 技术细节

### Analysis 命令
```bash
# 列出所有分析
spec-graph analysis --phase list

# 查看特定阶段分析
spec-graph analysis --phase propose

# 写入/更新分析
spec-graph analysis --phase propose \
  --content "关键决策..." \
  --tasks "T-001,T-002" \
  --docs ".spec-graph/artifacts/prd/PRD-001.md" \
  --templates "prd"
```

### Checklist 命令
```bash
# 生成质量检查清单
spec-graph checklist requirement/prd/PRD-001
```

### 机械检查
1. **需求引用**: 检查 story 是否引用了至少一个 requirement
2. **范围原子性**: 检查 story 是否足够小
3. **验收标准数量**: 检查是否有至少 2 个 AC
4. **需求解决状态**: 检查引用的 requirements 是否已完成
5. **路径安全**: 检查是否有超出项目范围的文件路径

### 软检查
1. **无模糊形容词**: 检查是否有 "fast"/"user-friendly" 等
2. **AC 可验证**: 检查每个 AC 是否可被测试验证
3. **边界情况**: 检查是否考虑了边界情况
4. **依赖声明**: 检查是否声明了依赖
5. **超出范围**: 检查是否明确列出了超出范围的项
